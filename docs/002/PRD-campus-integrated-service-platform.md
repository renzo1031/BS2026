# 大学生一体化服务平台项目规格

## 1. 大白话定位

这不是一个只会登记标题和备注的通用工单页面，而是学校学生事务的网上办事平台。学生可以办理宿舍报修、在读证明和活动场地申请；对应部门只能处理本部门业务；管理员维护账号和基础数据并查看审计记录。每类事项都有自己的表单字段、校验规则、处理结果和可验证的异常场景。

系统形态为前后端分离 Web 应用，面向桌面浏览器并兼顾常见移动宽度。论文题目建议使用“基于 Spring Boot 与 Vue 的高校学生事务一站式办理平台设计与实现”，避免把范围泛化为全部校园业务。

## 2. 范围判断

### 采纳

- 三类事项的专属字段、后端校验和完整状态流转。
- 学生本人、部门、全局三种数据范围。
- 在读证明编号和可下载的电子证明结果。
- 场地容量、起止时间和并发冲突校验。
- 三个业务部门的处理账号、处理详情和操作留痕。
- 管理员创建部门人员、停用账号、上下架事项和停用场地。

### 暂缓

- 学校统一身份认证、短信/微信通知、正式电子签章、多校区和移动 App。
- 报修图片上传。当前报修通过文字描述完成闭环，避免在没有对象存储和病毒扫描条件时增加不安全的上传入口。

### 不采纳

- 通用可视化流程引擎、AI 助手、支付商城、大屏驾驶舱、校园社交和教务成绩功能。

## 3. 角色与权限

| 能力 | 学生 STUDENT | 部门人员 STAFF | 管理员 ADMIN |
| --- | --- | --- | --- |
| 注册、登录、个人资料 | 本人 | 登录、个人资料 | 登录、个人资料 |
| 服务目录 | 查看启用事项和可用场地 | 查看 | 查看 |
| 新建申请 | 仅本人 | 禁止 | 禁止 |
| 查看申请 | 仅本人 | 仅本部门 | 全部 |
| 取消、评价、下载证明 | 仅本人且状态允许 | 禁止 | 禁止 |
| 受理、审核、驳回、办结 | 禁止 | 仅本部门 | 全部，操作留痕 |
| 统计 | 本人列表代替统计 | 仅本部门 | 全局 |
| 用户、事项、场地和日志 | 禁止 | 禁止 | 全局 |

权限必须同时由路由/按钮和后端 `Service` 数据范围校验实现。管理员可处理业务用于应急兜底，但不会伪装成部门人员，操作日志保留真实操作者。

## 4. 身份与登录

- 学生注册时，姓名和学号必须与初始化的学生身份名册匹配，且一个学号只能绑定一个账号。
- 部门人员由管理员创建并绑定一个部门，不能自行注册。
- 密码使用 BCrypt；数据库密码和 JWT 密钥只从环境变量读取。
- JWT 包含用户令牌版本并设置过期时间。退出时递增服务器端令牌版本，使旧令牌立即失效；停用账号也会在鉴权阶段被拒绝。
- 登录连续失败达到阈值后短时锁定，错误信息不区分账号不存在和密码错误。

## 5. 业务状态机

所有状态转换由后端 `Service` 校验，不能跳过步骤。

| 当前状态 | 操作角色 | 操作 | 下一状态 | 前置条件和副作用 |
| --- | --- | --- | --- | --- |
| SUBMITTED | 学生本人 | 取消 | CANCELLED | 记录取消；场地时段释放 |
| SUBMITTED | 本部门 STAFF / ADMIN | 受理 | ACCEPTED | 绑定处理人和受理时间 |
| ACCEPTED | 学生本人 | 取消 | CANCELLED | 尚未进入处理；记录取消 |
| ACCEPTED | 本部门 STAFF / ADMIN | 审核通过 | PROCESSING | 审核意见必填；场地申请继续占用时段 |
| ACCEPTED | 本部门 STAFF / ADMIN | 驳回 | REJECTED | 驳回理由必填；场地时段释放 |
| PROCESSING | 本部门 STAFF / ADMIN | 办结 | FINISHED | 结果必填；证明业务生成证明编号和验证码 |
| FINISHED | 学生本人 | 评价 | EVALUATED | 评分 1 到 5；每个申请只能评价一次 |

终态为 `REJECTED`、`CANCELLED` 和 `EVALUATED`。`FINISHED` 是可评价、可下载证明的业务完成态。

## 6. 三条核心流程

### F1 宿舍报修

学生填写故障地点、故障类型、紧急程度、标题和描述；后勤人员查看详情后受理、开始处理并填写维修结果；学生查看结果并评价。地点、故障类型和紧急程度为后端必填规则。

### F2 在读证明

学生填写证明类型、用途、语言、份数和领取方式；学生事务中心受理并审核；办结时系统生成证明编号和校验码。在线领取的学生可从本人详情下载可打印电子证明，其他用户不能下载。

### F3 活动场地申请

学生填写活动名称、场地、开始/结束时间、参加人数和联系人；系统校验未来时间、结束晚于开始、人数不超过容量以及同场地时段不重叠。团委人员受理、审核和办结；取消或驳回后释放时段。创建冲突检查在事务内锁定场地记录，避免并发重复占用。

## 7. 功能模块

1. 身份认证：名册注册、登录、退出、当前用户、登录失败限制。
2. 服务目录：分类、事项、可用场地。
3. 学生办理：三类动态表单、我的申请、详情、取消、评价、证明下载。
4. 部门处理：本部门分页筛选、详情、受理、审核、驳回、办结、部门统计。
5. 平台管理：创建部门人员、启停账号、启停事项、启停场地、全局统计。
6. 消息与审计：状态通知、完整流转记录和管理员操作日志。

## 9. 数据库概要

| 表 | 用途 | 关键约束 |
| --- | --- | --- |
| `sys_department` | 三个处理部门 | 部门名唯一 |
| `student_identity` | 可注册学生名册 | 学号唯一、绑定用户唯一 |
| `sys_user` | 用户与令牌版本 | 用户名、学号唯一；部门外键 |
| `sys_role`、`sys_user_role` | RBAC 角色 | 用户角色组合唯一 |
| `service_category` | 服务分类 | code 唯一 |
| `service_item` | 三类服务事项 | code 唯一；绑定部门和类型 |
| `venue` | 活动场地及容量 | 状态和容量约束由服务校验 |
| `service_request` | 申请主数据及三类专属字段 | 申请号唯一、版本号乐观锁、按本人/部门/状态索引 |
| `process_record` | 状态变化历史 | 申请和操作者外键 |
| `notice` | 站内通知 | 用户/已读索引 |
| `feedback` | 学生评价 | request_id 唯一、评分范围检查 |
| `operation_log` | 关键操作审计 | 用户/时间索引 |

系统共 13 张有实际调用方的表，不保留没有实现入口的附件、菜单权限占位表。事项专属数据放在申请主表中，以当前固定三类流程降低无意义联表复杂度；若将来事项类型扩展到十种以上，再迁移为扩展表或表单定义模型。

## 10. API 概要

- 认证与资料：`POST /auth/register`、`POST /auth/login`、`POST /auth/logout`、`GET /auth/me`、`PUT /profile`。
- 目录：`GET /catalog/categories`、`GET /catalog/items`、`GET /catalog/venues`。
- 学生申请：`POST /requests`、`GET /requests/my`、`GET /requests/{id}`、`POST /requests/{id}/cancel`、`POST /requests/{id}/feedback`、`GET /requests/{id}/certificate`。
- 部门处理：`GET /admin/requests`、`POST /admin/requests/{id}/accept`、`POST /admin/requests/{id}/approve`、`POST /admin/requests/{id}/finish`、`GET /admin/stats`。
- 管理能力：`GET /admin/users`、`POST /admin/staff`、`PATCH /admin/users/{id}/status`、`GET /admin/departments`、`GET /admin/service-items`、`PATCH /admin/service-items/{id}/enabled`、`GET /admin/venues`、`PATCH /admin/venues/{id}/status`、`GET /admin/logs`。
- 通知：`GET /notices/my`、`POST /notices/{id}/read`。

统一响应为 `{ code, message, data }`，同时返回正确 HTTP 状态。分页参数 `page >= 1`、`1 <= size <= 100`，时间使用 ISO-8601 本地时间。400 表示参数或非法状态，401 表示未登录/令牌失效，403 表示角色或数据越权，404 表示资源不存在，409 表示学号绑定、重复评价、并发状态或场地冲突。

## 11. 架构与版本

- 后端：Java 17、Spring Boot 3.5.9、MyBatis-Plus 3.5.14、MySQL、JWT、BCrypt、Maven。
- 前端：Vue 3、Vite、Vue Router、Pinia、Axios、Naive UI。
- 选用 Naive UI 是因为现有项目已完成接入且覆盖表单、表格、弹窗、消息和响应式布局；没有证据表明换成 Element Plus能解决业务问题，因此不换库。
- Controller 只处理协议和校验；状态、权限、场地并发和证明交付规则位于 Service；Mapper 只执行参数化查询。
- MyBatis-Plus 使用分页和乐观锁插件。场地创建额外使用数据库行锁，把“检查冲突 + 新增申请”放在同一事务。
- 前端只提供体验层角色守卫，后端鉴权和数据范围才是安全边界。

当前版本和配置已通过 Context7 的 Spring Boot 3.5.9、MyBatis-Plus 官方文档复核。行业调研网络调用在本轮返回 404，学校实际审批材料、证明版式和场地时段规则仍是未验证假设，后续接入真实学校时必须由业务老师确认。

## 12. 可追踪验收标准

| 编号 | 验收条件 |
| --- | --- |
| AC-01 | 名册内未绑定学生可注册；重复学号、姓名不匹配和非名册学生被拒绝。 |
| AC-02 | 正确密码登录成功；错误密码、未登录、过期或退出后的令牌返回正确 401。 |
| AC-03 | 学生只能创建和查看本人申请，不能访问他人申请或管理员接口。 |
| AC-04 | STAFF 只能查看、处理和统计本部门申请；跨部门操作返回 403。 |
| AC-05 | 宿舍报修按专属字段提交，完整经过受理、处理、办结和评价；非法跳步被拒绝。 |
| AC-06 | 在读证明按专属字段提交，办结后生成编号和验证码，本人可下载，其他用户被拒绝。 |
| AC-07 | 场地人数超容量、非法时间和重叠时段被拒绝；取消或驳回后同一时段可再次申请。 |
| AC-08 | 重复评价、重复受理、重复审核和并发版本冲突被后端拒绝。 |
| AC-09 | 管理员可创建部门人员、启停账号/事项/场地，普通角色无权调用。 |
| AC-10 | 列表分页、状态筛选、空数据和参数边界行为正确。 |
| AC-11 | 每次关键状态变化都有流转记录、通知和操作日志，响应不泄露密码哈希或异常堆栈。 |
| AC-12 | SQL 可在空库执行；后端测试、前端生产构建、三流程 API smoke 和桌面/移动 Playwright 验证通过。 |
