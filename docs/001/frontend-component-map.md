# 校园失物招领中心前端组件边界

## 1. 调研与取舍

- Vue 3 采用 Composition API 与 `<script setup lang="ts">`。
- Vue Router 使用路由 `meta.requiresAuth` 与 `meta.roles` 控制页面入口。
- Pinia 统一保存 token、当前用户和角色，避免各页面各自维护登录态。
- Element Plus 用于表单、表格、分页、详情抽屉和后台布局，减少自造基础组件。

## 2. 组件与页面边界

| 类型 | 文件/目录 | 职责 |
|---|---|---|
| 应用入口 | `App.vue` | 只承载路由出口，不写业务逻辑 |
| 公共布局 | `layouts/PublicLayout.vue` | 前台导航、登录状态、公共页面容器 |
| 用户布局 | `layouts/UserLayout.vue` | 个人中心侧边栏和子路由 |
| 后台布局 | `layouts/AdminLayout.vue` | 后台菜单、角色边界和子路由 |
| API 层 | `api/http.ts`, `api/modules.ts` | Axios 封装和接口函数 |
| 状态层 | `stores/auth.ts` | 登录、登出、当前用户、角色判断 |
| 前台页面 | `views/public/*` | 首页、列表、详情 |
| 用户页面 | `views/user/*` | 发布、我的发布、我的认领、我的线索、通知 |
| 后台页面 | `views/admin/*` | 物品审核、全量管理、认领、线索、用户、分类地点、公告、日志 |

## 3. 数据流规则

- 页面组件负责拉取数据和组合区域。
- 表格、表单暂不拆成过多子组件，避免毕业设计项目过度抽象。
- 登录态只从 Pinia 读取，不在 local component 重复存储。
- 所有后端权限仍以后端返回为准，前端隐藏菜单只做体验层。

## 4. 自审

- 需要：前端页面较多，没有组件边界会很快变成单文件堆代码。
- 必要：路由权限、登录态和后台布局必须先定，否则后续页面会反复返工。
- 完整性：覆盖前台、个人中心、后台和 API/Store 基础层。
- 不做：不做复杂低代码后台、不做大屏、不做动画堆砌。
